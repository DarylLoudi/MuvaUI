── CARA BUILD ──────────────────────────────────

  Setiap kali kamu edit file di src/, jalankan:

    python3 build.py

  Ini akan concat semua file Lua menjadi satu MuvaUI.lua.
  Output akan muncul di terminal seperti ini:

    [OK] Built MuvaUI.lua — 9/40 files, 40.9 KB
         >> C:\Users\daryl\Downloads\MuvaUI\MuvaUI.lua

  Catatan: file yang belum dibuat akan di-skip dengan peringatan [WARN].
  Itu normal selama komponen belum diimplementasi semua.


── CARA TEST DI ROBLOX (EXECUTOR) ─────────────

  1. Buka Roblox game di executor kamu (Synapse, KRNL, dll)

  2. Upload MuvaUI.lua ke GitHub raw atau hosting lain,
     lalu jalankan di executor:

       local MuvaUI = loadstring(game:HttpGet("https://raw.githubusercontent.com/USERNAME/REPO/main/MuvaUI.lua"))()

  3. Atau untuk test lokal tanpa upload, pakai file executor
     yang support readfile():

       local MuvaUI = loadstring(readfile("C:/Users/daryl/Downloads/MuvaUI/MuvaUI.lua"))()


── CARA TEST DI ROBLOX STUDIO (OPSIONAL) ───────

  Catatan: MuvaUI dirancang untuk executor (loadstring + CoreGui),
  bukan ModuleScript. Tapi bisa di-test di Studio dengan:

  1. Buka Script di StarterPlayerScripts
  2. Copy-paste isi MuvaUI.lua ke atas script, lalu pakai langsung:

       local Window = Library:CreateWindow({ Title = "Test" })


── CARA AUDIT VISUAL ───────────────────────────

  Buka mockup.html di browser untuk melihat semua komponen:

    C:\Users\daryl\Downloads\MuvaUI\mockup.html

  Ini adalah source of truth visual sebelum implementasi Lua.
  Semua komponen, warna, dan behavior di Lua harus cocok dengan mockup.
